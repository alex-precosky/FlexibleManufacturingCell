 Glossary
* Note: Actions specified more thouroughly in class diagram document, describing them as C functions including parameters.
Actor
 Description
 CA
 Event/Action
 Description
BarcodeReader
 Reads information from the barcode of entering blanks
 ReadBarcode
 Action
 Retreives the information read by the barcode reader.  Waits until the information has scanned and has appeared in memory before returning.
Blank
 Blanks that goes through the system.  It stores its own barcode's information, i.e. the processes it needs to go through, the time to spend in each, and its maximum allowed tiem in the system.
 New
 Action
 Tell the blank its processing order for internal storage.  Basically tell the blank what its barcode says.
BlankBuffer
 A sync object for holding blank information.
 putBlank
 Action
 Place a blank in the buffer
getBlank
 Action
 Retrieve a blank from the buffer
FeedBelt
 Carries new blanks into the system and up to the barcode reader
 turn_on
 Action
 Start the motion of the feed belt
turn_off
 Action
 Halt the motion of the feed belt
CraneController
 The controller for the overhead travelling cranes for moving blanks.  Manages cranes reservations.
 move_xy
 Action
 Move the requested crane to the requested coordinate (x,y)
request
 Action
 Inform the cranes that we need one for a job.  Wait until it is done its current job.
signal_done
 Action
 Inform the crane that it is finished its current job and that it can go do other jobs if it needs to.
DepositBeltSensor
 Senses when a blank is at the beginning of the deposit belt.
 IsFree
 Action
 Poll to see if we can put something on the deposit belt.
FeedbeltSensor
 Detects blanks in front of the barcode reader.
 SIGALRM
 Event
 The light sensor picks up a blank in position in front of the barcode reader.
ProcessUnitController
 The controller for the processing units.  Manages which units are reserved at any given time.
 reserve
 Action
 Tell processing units which ones we want to use.  Block until these units are no longer needed by other blanks.
turn_on
 Action
 Activate a processing unit
signalDone
 Action
 Tell the controller that the blank is done with the system so that it can let other blanks in the need the units the blank had reserved.
